# Relatórios do Projeto

Esta pasta reúne os documentos de análise e especificação da Plataforma de Anúncios de Estágios Acadêmicos. Os arquivos servem como base para entender o problema, definir o escopo inicial, registrar requisitos e orientar a implementação do sistema.

## Função de cada arquivo

- `documento-visao.md`: apresenta a visão geral do produto, descrevendo o problema, o objetivo da plataforma, o público-alvo, os benefícios esperados, o escopo inicial, premissas e restrições do projeto.

- `requisitos-software.md`: registra os requisitos do sistema. Inclui requisitos funcionais, requisitos não funcionais, regras de negócio, critérios básicos de aceitação e pendências de refinamento.

- `user-stories.md`: descreve as histórias de usuário e seus critérios de aceitação. Esse arquivo ajuda a transformar as necessidades dos usuários em funcionalidades que podem ser implementadas e validadas pela equipe.

- `requisitos.pdf`: versão em PDF da documentação de requisitos. Serve como artefato de entrega ou consulta em formato fechado, útil para apresentação, revisão ou envio externo.

- `diagrama_casos_de_uso_estagios.png`: imagem do diagrama de casos de uso do sistema. Representa visualmente os atores e as principais interações esperadas entre usuários e plataforma.
