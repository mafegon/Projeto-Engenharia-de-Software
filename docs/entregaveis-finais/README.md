# Projeto Integrador

Esta pasta reúne os documentos entregáveis do projeto final.

## Função de cada arquivo

- [`documento de requisitos`](documento-de-requisitos.md): contém a elicitação e especificação completa dos requisitos do sistema.
  
- [`gestao de processo.`](gestao-de-processo.md): evidência de que foi usado um processo de desenvolvimento.

- [`gestao de arquitetura e design`](arquitetura-e-design.pdf): decisões arquiteturais e de design que guiam a implementação.

- [`UML`](UMl.pdf): modelagem visual (casos de uso, classes e sequência) que detalha a arquitetura lógica e a dinâmica de interação entre alunos, empresas, vagas e candidaturas, junto da especificação expandida dos casos de uso.

- [`relatório de testes de software plano de testes`](relatorio-de-testes-de-software-plano-de-testes.pdf): apresenta casos de teste executados, relacionando sua classificação, descrição funcional, parâmetros de entrada e resultados esperados.

- [`testes unitários e de integração automatizados`](testes-unitarios-e-de-integracao-automatizados.pdf): detalha os testes unitários implementados para validar as regras de negócio do sistema.
  
- [`relatório de cobertura`](relatorio-de-cobertura.pdf): apresenta os resultados da análise de cobertura de código das principais classes da aplicação.
